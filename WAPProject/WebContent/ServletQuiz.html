Servlet Quiz

1. What is the difference between a web server and a web container?
Web Server: A server which is capable of handling requests, sent by a client and respond back with a HTTP responce.

Web Container is used to manage the components like servlets, JSP. It is a part of the web server.


2. What is servlet?
A servlet is server-side java code that can handle http requests and return dynamic content - Servlets are managed by a servlet engine or container. Each request that comes in results in the spawning of a new thread that runs a servlet (eliminating the cost of creating a new process every time). One of the best known open source servlet contaner is Tomcat.


3. How do web servers and web containers interact with servlets?
Servers that support servlets have as a helper app a servlet container.
When a request comes to the web server, if the server sees the request is for a servlet, it passes the request data to the servlet container.
The servlet container locates the servlet, creates request and response objects and passes them to the servlet, and returns to the web server the response stream that the servlet produces.
The web server sends the response back to the client browser to be rendered.


4. Who creates request objects?
The servlet container create request and response objects and passes them to servlets.


5. What are the states in the servlet lifecycle?
There are 6 stated in the servlet lifecycle.
  1. Load servlet class (Class c = Class.forname(...))
  2. 2. Instantiate servlet (c.newInstance());
  3. 3. init() called only once in the servlet's life. Must complete before Container can call service().
  4. 4. service() (called for each request, each request runs in a separate thread)
  5. 5. destroy() (called only once)


6. Who calls init and when?
init() is called first by the servlet container to indicate to a servlet that the servlet is being placed into service.


7. Which of init, service, and doGet should you override?
init(): If you have initialization code(like getting a database connection or registering yourself with other objects), then you'll override the init() method in the servlet class.

service(): We can not override the service() method. Service implementation from HTTPServlet worry about calling the right one.

doGet() or doPost(): Whicever one(s) you override tells the container what you support. If you don't override doPost(), for example, then you're telling the container that this servlet does not support HTTP post requests.

8. In what sense are servlets multi-threaded?
Servlets are intrinsically multithreaded. This meand a single instance can be assessed by more than one thread. if the container received multiple requests for one servlet simultaneously, the service() method of that servlet will be executed concurrently in multiple threads.


9. What are the implications of this for servlet instance variables?
Due to the thread safety in servlet it is advised to not make instance variables. SInce servlet lifetime is controlled by the server so the duration of a servlet isntance is uncertain, as may be the number of instance and the number of thread involed/
